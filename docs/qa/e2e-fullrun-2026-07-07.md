## 新发现 Bug 详情

### BUG-018 🟡 中 — 切 OrgTree tab 后 React Flow 节点不渲染
- **现象**：点 OrgTree tab 后，**OrgTree 区完全空白**（无 CEO/HR/QA 节点）
- **但**：OrgTree 内的 `+` 按钮（Create Agent）能点开 modal —— 间接证明 OrgTree 容器在
- **推测**：
  - React Flow `<ReactFlow nodes={...}>` 节点未传入或为空
  - 切 tab 时 OrgTree 组件 unmount，节点状态丢失
  - 你的 `App.tsx +42` 或 `store.ts +12` 改了渲染逻辑
- **位置**：`apps/web/src/components/OrgTree.tsx` 或 `App.tsx`
- **影响**：**用户看不到组织结构图**——核心 UI 不可用

### BUG-019 🟠 中高 — CEO 在大型 workspace 上启动慢/卡死
- **现象**：
  - sandbox（.e2e-sandbox/ 3 文件）下 CEO 30s 内响应
  - PoE2LI（100+ 文件）下 CEO 35s+ 才出 chat history，T+90s 仍只有 user 消息
- **推测**：
  - agent 启动时 `list_files` 读所有文件
  - LLM prompt 拼所有 file path → **token 超限**
  - 或 LLM provider 限流
- **位置**：`agents/agent.py` 启动逻辑
- **建议**：
  - 限制 `list_files` 深度（默认 maxdepth=2）
  - 加 `.hiveweaveignore` 类似 .gitignore
  - workspace 大于 N 文件时分批加载
- **影响**：在真实大项目上**用户首次启动需要等 1-2 分钟**，且容易超时
