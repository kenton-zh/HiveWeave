### BUG-031: FolderPicker 地址栏无法直接输入/粘贴路径

**严重度**: 中
**位置**: `apps/web/src/components/FolderPicker.tsx:132-152`
**现象**: 地址栏是 `<div onClick={() => setAddressEditing(true)}>`, 点击后变成 `<input>`, 但:
- 没有 paste 监听, 用户粘贴路径后默认行为可能丢失
- 必须双击文件夹才能进子目录(没有 onClick 单击高亮)
- MCP 测试工具 `browser_click` 的 `doubleClick` 布尔参数被 reject, 真实用户双击仍可工作

**复现**:
1. 打开 FolderPicker
2. 试图在地址栏粘贴 `D:\PC_AI\Project\PoE2LI` 后按 Enter
3. 不会跳转 (粘贴不一定生效)
4. 必须双击目录列表中的文件夹一层层进入

**影响**: 用户/测试工具需要多次双击才能定位到目标 workspace

### BUG-034: 前后端 chat_messages 字段命名不一致 (核心根因)

**严重度**: **高 (阻塞"团队信息"功能)**
**位置**:
- 后端: `apps/hiveweave-py/src/hiveweave/services/chat_message.py:246-252` (输出 snake_case)
- 前端: `apps/web/src/components/ChatPanel.tsx:160-177` (读 camelCase)

**现象**:
- 后端 `_row_to_msg()` 输出 `is_background` / `is_streaming` / `is_read` / `is_context` / `team_from_agent_id` / `team_to_agent_id` (snake_case)
- 前端 `mapDbToChatMessages()` 读 `m.isBackground` / `m.isStreaming` / `m.isRead` / `m.isContext` / `m.teamFromAgentId` / `m.teamToAgentId` (camelCase)
- **6 个字段全部对不上** → 前端 `isTeamChannelMessage()` 永远返回 false → `teamMessages.length === 0` → "团队沟通"折叠区永不显示

**用户痛点**: CEO 跟 HR/QA 之间的对话 (招聘请求、状态汇报) 全部显示在主 Chat 面板, 用户感觉"团队信息都发到了 CHAT 里"

**修复** (二选一, 5 分钟活):
- 方案 A (后端): `_row_to_msg` 额外加 camelCase 别名
  ```python
  d["isBackground"] = d["is_background"]
  d["isStreaming"] = d["is_streaming"]
  d["isRead"] = d["is_read"]
  d["isContext"] = d["is_context"]
  d["teamFromAgentId"] = d.get("team_from_agent_id")
  d["teamToAgentId"] = d.get("team_to_agent_id")
  ```
- 方案 B (前端): `mapDbToChatMessages` 改读 snake_case

**推荐方案 A** (后端单点修改, 不影响其他 8 个 API 端点)

### BUG-035: 后端 `/api/*` 持续返回 502 Bad Gateway

**严重度**: 高
**位置**: uvicorn / FastAPI 进程
**现象**:
- 进程 31932 仍在跑, TCP 4000 端口通
- 但所有 `/api/*` 返回 502 (projects / org/tree / chat/messages 等全部失败)
- WebSocket 仍能推送旧数据 (前端还能看到缓存的旧消息)
- 重启后端可能能恢复

**推测根因**: uvicorn 单 worker 模式被长连接 (LLM streaming + 11 个 agent 持续 in-context) 阻塞, 新 HTTP 请求进不了 worker

**临时方案**: 重启后端
**长期方案**: uvicorn 配置多 worker / async 优化
